## 🎮 Возможности

- **Inline-режим** — не нужно добавлять бота в чат, просто вызовите через `@username_бота`
- **Групповой режим** — добавьте бота в группу, и заработают команды (`/grow`, `/top`, …) и авто-события (писюн дня в полночь). Накопленная inline-статистика подключается автоматически
- **Разделение по чатам** — данные хранятся отдельно для каждой группы
- **Дуэли** — вызовите участников на дуэль; вызов сгорает по таймауту, если его не приняли
- **Казино** — поставьте сантиметры на удачу (шанс 50%)
- **Писюн дня** — случайный участник получает бонус
- **Топ участников** — общий рейтинг и топ прироста за неделю
- **Достижения** — открываются за рост, серии и победы
- **Статистика** — размер, рекорд, серия роста, винрейт в дуэлях

## 📦 Установка

### 1. Создайте бота в Telegram

1. Откройте [@BotFather](https://t.me/BotFather) и отправьте `/newbot`
2. Укажите имя и username (должен заканчиваться на `bot`)
3. Скопируйте токен вида `123456789:ABCdef...`

### 2. Включите Inline Mode

1. В [@BotFather](https://t.me/BotFather) отправьте `/mybots` → выберите бота
2. **Bot Settings** → **Inline Mode** → **Turn on**

> Бот работает только на inline-запросах и кнопках, поэтому включать
> «Inline feedback» не требуется.

### 3. Клонируйте репозиторий и настройте `.env`

```bash
git clone <url-репозитория>
cd FairGrowerBot
cp .env.example .env   # затем впишите BOT_TOKEN
```

```
BOT_TOKEN=ваш_токен_от_botfather
BOT_USERNAME=@FairDickGrowerBot
```

> ⚠️ `.env` и `*.db` добавлены в `.gitignore`.

### 4. Установите зависимости

```bash
python -m pip install -r requirements.txt
```

## 🖥️ Запуск

### Локально

```bash
python main.py
```

### Docker

```bash
docker build -t fairgrowerbot .
docker run -d --name fairgrowerbot \
  -e BOT_TOKEN=ваш_токен \
  -v fairgrower_data:/data \
  --restart unless-stopped \
  fairgrowerbot
```

База данных сохраняется в volume `/data`, поэтому переживает перезапуск контейнера.

### Обновление на сервере (с сохранением базы)

Скрипт `deploy/upgrade.sh` делает всё безопасно: бэкап базы → остановка бота
(systemd-сервис или процесс `python main.py`) → `git pull` → установка
зависимостей → миграция БД с проверкой, что записи не потерялись → запуск
обратно. При сбое до успешной миграции база восстанавливается из бэкапа.

```bash
cd /path/to/FairGrowerBot
./deploy/upgrade.sh              # обновить текущую ветку
./deploy/upgrade.sh main        # или указать ветку
SERVICE=fairdickbot ./deploy/upgrade.sh   # если сервис не определился сам
```

### systemd (автозапуск)

Проще всего — скрипт установки: он подставит в шаблон `deploy/fairdickbot.service`
вашего пользователя, путь к репозиторию и интерпретатор (venv, если есть),
поставит юнит и включит автозапуск:

```bash
cd /path/to/FairGrowerBot
sudo ./deploy/install-service.sh
```

После этого `deploy/upgrade.sh` сам увидит сервис и будет им управлять.

Управление:

```bash
sudo systemctl status fairdickbot
sudo systemctl restart fairdickbot
journalctl -u fairdickbot -f        # логи
```

Шаблон юнита лежит в `deploy/fairdickbot.service` — при желании его можно
заполнить и установить вручную.

## 📖 Описание команд

Есть два способа играть:

- **Inline** — вызовите `@username_бота` в любом чате и выберите команду из меню.
- **В группе** — добавьте бота в группу участником; тогда работают команды
  `/grow`, `/top`, `/weektop`, `/dickofday`, `/stats`, `/duel`, `/casino`, а
  «писюн дня» выбирается автоматически в полночь. При добавлении бот пришлёт
  приветствие с кнопкой «Активировать чат» — она подключает статистику, уже
  наигранную в inline-режиме (склейка `chat_instance ↔ chat_id`).

> Для авто-событий нужен установленный extra `python-telegram-bot[job-queue]`
> (уже прописан в `requirements.txt`).

#### 📈 Grow
Изменяет размер на случайное значение от **-10 до +35 см**. Раз в сутки (по времени из `TIMEZONE`, по умолчанию МСК). Ведёт серию дней подряд. Даёт опыт персонажу.

#### 👤 Профиль
Глобальный RPG-персонаж (один на аккаунт, общий для всех чатов): класс, уровень, опыт и характеристики (💪 Сила, 🛡️ Живучесть, 🍀 Удача). Опыт капает с grow, дуэлей и экспедиций. При первом открытии предлагается выбрать один из трёх классов — он задаёт распределение характеристик. Сила повышает шанс победы в дуэлях (в коридоре 35–65%).

#### 🗺️ Экспедиция
Отправьте героя в зону (открываются по уровню) — по возвращении он приносит опыт, монеты и предмет. Награда ждёт, пока её заберут; в группе бот сам объявит возврат (дайджестом). Добыча делится на 7 редкостей от ⚪ Обычного до 🌟 Реликтового — Удача повышает шанс редкого лута.

#### 🎒 Инвентарь
Найденные предметы (⚔️ оружие, 🛡️ броня, 💍 артефакт). Экипировка усиливает характеристики персонажа и, соответственно, влияет на дуэли.

#### 🐉 Босс
У чата появляется босс с общим запасом HP (сам днём или по команде `/boss`). Участники бьют его по очереди (кулдаун между ударами), урон зависит от Силы и экипировки. После победы монеты и лут раздаются по вкладу урона, а нанёсшему больше всех — повышенный шанс редкой добычи.

#### 🏰 Подземелье
Push-your-luck забег за монеты (`/dungeon`): платите за вход и спускаетесь глубже. Каждый шаг — либо сундук (монеты + будущий лут), либо ловушка (урон, зависит от глубины). Живучесть повышает запас HP. Можно уйти с накопленной добычей в любой момент — или погибнуть и потерять всё, включая плату за вход.

#### 🏪 Магазин
Трата монет (`/shop`): сундуки (случайный предмет, у дорогих выше шанс редкого) и платная смена класса. Экран редактируется на месте, покупки показываются всплывающим окном — чат не засоряется.

#### 🏡 Ферма
Пассивный доход (`/farm`): купите ферму (🌱 Грядка → 🪴 Теплица → 🏡 Плантация) и собирайте монеты, которые копятся со временем. Накопление ограничено сутками дохода — выгодно заглядывать и собирать.

#### 🏆 Top
Топ-10 участников чата по текущему размеру.

#### 📅 Топ недели
Топ-10 по суммарному приросту за последние 7 дней.

#### 🎉 Dick Of Day
Случайный участник получает бонус **+1..+30 см**, раз в сутки. Повторный вызов показывает уже выбранного.

#### 📊 Stats
Личная статистика: текущий размер, рекорд, серия роста, победы/поражения и винрейт в дуэлях, число открытых достижений.

#### ⚔️ Duel
Создаёт вызов со ставкой (**1–5 см**, не больше вашего размера). Любой участник может принять кнопкой; исход — 50/50. Непринятый вызов сгорает через 2 минуты.

#### 🎰 Casino
Ставка **1–10 см**: с шансом 50% удваивается, иначе сгорает.

#### 🏅 Достижения
Открываются автоматически: первый рост, 50 и 100 см, джекпот (+35), неудача (-10), серия 7 дней, победы в дуэлях, писюн дня.

## 🛠️ Разработка

```bash
python -m pip install -r requirements-dev.txt
ruff check .        # линтер
pytest -q           # тесты
```

Тесты работают на изолированной временной БД и не требуют токена. CI (GitHub Actions) прогоняет ruff и pytest на каждый push и PR.

## 📁 Структура проекта

```
FairGrowerBot/
├── main.py            # Точка входа: логирование, обработчики, error handler
├── handlers.py        # Inline-меню, кнопки, команды, дуэли, казино
├── group.py           # Групповой режим: команды, добавление в чат, авто-события
├── database.py        # SQLite: схема, миграции, атомарные операции, связка чатов
├── utils.py           # Время (часовой пояс) и форматирование упоминаний
├── config.py          # Константы и настройки
├── ROADMAP.md         # План развития в RPG
├── requirements.txt   # Зависимости для запуска
├── requirements-dev.txt
├── pyproject.toml     # Настройки ruff и pytest
├── Dockerfile
├── tests/             # Тесты слоя данных и утилит
└── .github/workflows/ci.yml
```
